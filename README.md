## 🧑‍💻 프로젝트 소개
이 저장소는 오픈소스소프트웨어 강의에서 진행한 다양한 프로젝트들을 하나로 통합한 레포지토리입니다.  
각 프로젝트는 Git Subtree를 활용하여 커밋 기록을 유지한 상태로 통합되었습니다.

## ⚙️ 개발자 관점에서의 학습 내용

### 1. Git & GitHub 활용
- Git 기본 명령어 (clone, add, commit, push)에 대한 이해
- Fork 및 협업 흐름(Pull Request 등)에 대한 개념 학습
- Git Subtree를 활용한 여러 레포지토리 통합 경험

### 2. 버전 관리의 중요성
- 코드 변경 이력을 체계적으로 관리하는 방법 이해
- 협업 시 발생할 수 있는 충돌 상황과 관리 방식 학습
- 개별 프로젝트와 통합 프로젝트 관리 방식의 차이 이해

### 3. 프로젝트 구조 설계 및 확장
- 여러 프로젝트를 하나의 저장소에서 관리하는 구조 설계
- 폴더 단위로 프로젝트를 분리하고 유지보수성을 고려한 구성
- 강의 이후 Git Subtree를 활용하여 프로젝트들을 하나로 통합하는 작업을 직접 수행  
  → 커밋 이력을 유지하면서 구조적으로 통합하는 실무적인 Git 활용 경험

### 4. 문제 해결을 통한 기술 이해
- 초기 환경 설정 문제로 인해 작업 내용이 저장되지 않는 오류 경험
- Git의 동작 방식과 저장 구조를 재이해하며 문제 해결
- 단순 사용을 넘어 Git 내부 동작에 대한 이해도 향상

## 🎓 학습자 관점에서의 성장

### 1. Git에 대한 이해도 향상
처음에는 Git을 단순한 코드 저장 도구로 생각했지만,  
강의를 통해 버전 관리와 협업을 위한 핵심 도구라는 점을 이해하게 되었습니다.

### 2. 시행착오를 통한 학습
강의 초반에는 표준 폴더 실습 및 Git 사용 과정에서 여러 오류를 겪었고,  
특히 설치 환경 문제로 인해 작업물이 정상적으로 저장되지 않는 어려움을 경험했습니다.  

이 과정에서 교수님께 질문을 반복하며 문제의 원인을 분석하고 해결하는 과정을 거쳤고,  
그 결과 Git 사용 흐름에 대한 이해가 점차 명확해졌습니다.

### 3. 문제 해결 능력 향상
단순히 명령어를 따라 하는 것이 아니라,  
에러 메시지를 해석하고 원인을 추론하여 해결하는 경험을 통해  
스스로 문제를 해결할 수 있는 능력을 기르게 되었습니다.

### 4. 자기주도 학습 경험
강의 이후에는 배운 내용을 바탕으로 Git Subtree를 추가적으로 학습하였고,  
여러 개의 프로젝트를 하나의 저장소로 통합하는 작업을 스스로 수행하였습니다.  

이를 통해 단순한 과제 수행을 넘어,  
기술을 스스로 확장하고 응용하는 경험을 할 수 있었습니다.

---

## 🚀 느낀 점

처음에는 Git 사용이 익숙하지 않아 많은 어려움을 겪었지만,  
여러 시행착오를 반복하면서 Git의 개념과 동작 원리를 점차 이해하게 되었습니다.  

특히 설치 환경 문제로 인해 작업이 저장되지 않았던 경험은  
Git의 동작 방식에 대해 다시 생각해보는 계기가 되었고,  
문제를 해결하는 과정에서 Git을 보다 정확하게 다룰 수 있게 되었습니다.  

또한 교수님께 지속적으로 질문하며 문제를 해결해 나가는 과정에서  
단순히 결과를 얻는 것이 아니라 **과정을 이해하는 학습의 중요성**을 깨닫게 되었습니다.  

강의에서 배운 내용을 기반으로, 종강 이후에는  
Git Subtree를 활용하여 여러 프로젝트를 하나의 저장소로 통합하는 작업까지  
스스로 수행할 수 있었으며,  

이를 통해 단순한 학습을 넘어  
**실제 개발 환경에서도 활용 가능한 수준으로 성장했음을 느낄 수 있었습니다.**
